v2.4.0: Bundle E — won't update, won't boot, disk's full

The everyday bench killers, all read-first and documented.

- pendingreboot.py — reads every pending-reboot signal (CBS, Windows Update,
  PendingFileRenameOperations, queued computer rename, ConfigMgr) and explains
  each in plain English; optional confirmed restart.
- wuhistory.py — Windows Update history via the Microsoft.Update.Session COM
  searcher, with the cryptic ResultCode/HResult values decoded into plain English
  and remediation; last successful scan/install; health of the WU services.
- repair.py — adds DISM AnalyzeComponentStore, StartComponentCleanup (and a
  clearly-warned /ResetBase), and a read-only reserved-storage check, all streamed
  through the existing JobStore repair-tool pattern.
- boottime.py — Diagnostics-Performance Event 100/101-110: boot durations, the
  specific apps/drivers/services Windows blamed, Fast Startup state, uptime, and a
  trend via history.py. Degrades to uptime/Fast-Startup without admin.

Bridge: pending_reboot, restart_now, wu_history, wu_health, boot_performance.
UI: two new Toolbox cards (pending restart, update doctor), an Event Log "Boot
